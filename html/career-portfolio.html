구현 근거

클라우드 관리 콘솔에서 실제로 맡은 구현 단위입니다.

Case 01

생성 폼과 제출 데이터

자원과 알람 유형에 따라 입력 항목과 검증이 달라지는 화면을 구현했습니다.

  • 유형 선택에 따라 입력 컴포넌트, 기본값, 검증 조건 분기
  • 생성·수정 모드에서 초기값과 제출 payload를 다르게 구성
  • 제출 이후 목록, 상세, 로그 확인까지 이어지는 상태 처리

Case 02

조회 화면과 서버 상태

조건 조합이 많은 로그·자원 화면에서 검색 조건과 API 응답을 맞췄습니다.

  • 기간, 심각도, 플랫폼, 자원 유형, 이름 검색 조건을 API 파라미터로 연결
  • 서버 페이지네이션, 재조회, 캐시, 로딩·빈 값·에러 상태 처리
  • 통계 카드, 차트, 목록, 상세 패널이 같은 조회 조건을 바라보게 구성

Case 03

공통 UI 품질 개선

테이블과 버튼 문제를 개별 화면의 임시 처리로 두지 않고 공통 컴포넌트로 개선했습니다.

  • 컬럼 리사이즈, 컬럼 표시 제어, 고정 컬럼, 정렬 상태를 개선
  • 검색 필터, 날짜 필터, 서버 페이지네이션 흐름을 공통 패턴으로 처리
  • 긴 텍스트, 링크형 셀, 상태 태그, 버튼 로딩 상태를 재사용 가능한 형태로 다듬음
